# 저장소 지침

## 프로젝트 구조와 모듈 구성

현재 추적된 저장소는 루트의 `README.md`, `LICENSE`, `AGENTS.md`를 중심으로 한 초기 문서 스캐폴드입니다. 구현 코드가 추가되면 `src/` 아래에 두고, 테스트는 `tests/` 아래 또는 소스 옆의 `*.test.*` 파일로 둡니다. 정적 자산은 런타임에 따라 `assets/` 또는 `public/`에 둡니다. 최상위 파일은 프로젝트 메타데이터, 문서, 설정, 잠금 파일로 제한합니다.

## 빌드, 테스트, 개발 명령

현재 추적된 브랜치에는 빌드 시스템이나 패키지 매니저 설정이 커밋되어 있지 않습니다. 도구 체인이 커밋되기 전에는 저장소 루트에서 다음 확인 명령을 사용합니다.

- `git status --short`: 변경 전후의 작업 트리 상태를 확인합니다.
- `git diff -- AGENTS.md README.md`: 문서 변경을 커밋 전에 검토합니다.
- `git log --oneline -n 5`: 최근 커밋 메시지 스타일을 확인합니다.

패키지 매니저나 빌드 도구가 커밋되면 실제로 실행 가능한 명령과 관련 잠금 파일을 이 섹션에 함께 문서화합니다.

## 코딩 스타일과 이름 규칙

파일과 디렉터리 이름은 간결하고 설명적으로 작성합니다. 문서와 자산 파일명은 `design-notes.md` 또는 `button-icons.svg`처럼 소문자 케밥 케이스를 선호합니다. 언어 스택이 커밋되면 해당 포매터와 린터를 따르고, 분명한 이유 없이 경쟁하는 스타일 도구를 추가하지 않습니다. Markdown 파일은 ATX 제목, 짧은 문단, 명령이나 코드 조각을 보여줄 때 언어 태그가 있는 fenced code block을 사용합니다.

## 테스트 지침

현재 추적된 브랜치에는 테스트 프레임워크가 커밋되어 있지 않습니다. 의미 있는 소스 코드와 함께 테스트를 추가하고, 단일 명령으로 쉽게 실행되게 유지합니다. 테스트 이름은 `popover-position.test.ts` 또는 `test_popover_position.py`처럼 검증하는 동작이 분명하게 드러나게 작성합니다. 테스트 프레임워크, 실행 명령, 커버리지 기대치가 생기면 이 섹션을 갱신합니다.

## 커밋과 Pull Request 지침

현재 히스토리는 짧으므로 자세한 규칙은 아직 확립되지 않았습니다. `Add popover positioning docs` 또는 `Create base UI module`처럼 짧고 명령형인 커밋 메시지를 사용합니다. Pull Request에는 간단한 요약, 검증 절차, 관련 이슈, UI에 보이는 변경의 스크린샷이나 녹화를 포함합니다.

## 에이전트 전용 지침

편집하기 전에 저장소 상태를 확인하고 관련 없는 로컬 변경을 덮어쓰지 않습니다. 프로젝트 구조, 도구, 워크플로가 바뀌면 이 가이드를 최신 상태로 유지합니다.
